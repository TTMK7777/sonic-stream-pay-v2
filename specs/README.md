# specs/ - Feature Specifications

## Naming Convention
```
NNN-feature-name.md
```

Example:
- `001-streaming-contract.md`
- `002-stream-list-events.md`
- `003-withdraw-ui.md`
- `004-mainnet-deployment.md`

## Specification Template

```markdown
# NNN: Feature Name

## Status
Draft | Review | Approved | Implemented

## Summary
One-line description.

## Requirements
- MUST: ...
- SHOULD: ...
- MAY: ...

## Design
Technical approach.

## Testing
How to verify.
```

## Priority Keywords
- **MUST** -- Required for the feature to function
- **SHOULD** -- Expected but not blocking
- **MAY** -- Optional enhancement
